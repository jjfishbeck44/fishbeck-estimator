# CLAUDE.md

## Project Overview

Fishbeck Estimator is an AI-powered construction cost estimator for Fishbeck Innovations LLC, a construction and property maintenance company in the Twin Cities, Minnesota. Users describe a project in plain text and receive a structured cost breakdown generated by Claude.

**Tech stack:** Vanilla JavaScript frontend, Node.js serverless backend (Vercel Functions), Claude API (Anthropic SDK), Upstash Redis for rate limiting.

## Architecture

```
Client (public/)  →  POST /api/estimate  →  api/estimate.js (handler)
                                                ├── lib/ratelimit.js  (Upstash Redis, 10 req/60s/IP)
                                                ├── lib/prompt.js     (system prompt + pricing data)
                                                └── lib/claude.js     (Claude API, 25s timeout)
```

**Request flow:** CORS preflight → method guard (POST only) → rate limit → input validation (non-empty, ≤1000 chars) → Claude API call → JSON response.

**Frontend state machine:** INPUT → LOADING → RESULTS / CLARIFICATION / ERROR

## API Contract

**Endpoint:** `POST /api/estimate`

**Request body:**
```json
{ "input": "string (1–1000 chars, project description)" }
```

**Success response (estimate):**
```json
{
  "status": "estimate",
  "clarification_message": null,
  "line_items": [{ "label": "string", "description": "string", "range_low": 0, "range_high": 0 }],
  "total_low": 0,
  "total_high": 0,
  "notes": "string or null",
  "out_of_scope": ["string"]
}
```

**Success response (clarification needed — input too vague):**
```json
{
  "status": "clarification_needed",
  "clarification_message": "string",
  "line_items": [],
  "total_low": 0,
  "total_high": 0,
  "notes": null,
  "out_of_scope": []
}
```

**Error responses** return `{ "error": "<code>", "message": "..." }` with these codes:

| Code | HTTP Status | Cause |
|------|-------------|-------|
| `method_not_allowed` | 405 | Non-POST request |
| `rate_limited` | 429 | >10 requests per IP per 60s |
| `missing_input` | 400 | Empty or missing `input` field |
| `input_too_long` | 400 | Input exceeds 1000 characters |
| `api_timeout` | 500 | Claude API took >25 seconds |
| `invalid_json` | 500 | Claude returned non-JSON response |
| `internal_error` | 500 | Unexpected server error |

## Testing Conventions

- **Framework:** Jest v29, Node environment
- **Pattern:** All tests in `tests/**/*.test.js`
- **Mocking:** All external dependencies are mocked (`@anthropic-ai/sdk`, Upstash). No live API calls in tests.
- **Helpers:** Tests use `makeReq()` / `makeRes()` factory functions for request/response objects
- **Coverage areas:** Happy paths, error states, rate limiting, input validation, timeout handling

## Code Conventions

### General
- **Plain JavaScript** — no TypeScript, no transpilation
- **CommonJS** (`require`/`module.exports`) on the backend
- **IIFE pattern** with `'use strict'` on the frontend
- **No linter or formatter configured** — match the existing code style
- **Semicolons used consistently** — all source files use semicolons, follow the same convention

### Naming
- `camelCase` for functions and variables
- `UPPER_SNAKE_CASE` for constants (`STATES`, `TIMEOUT_MS`, `MAX_INPUT_LENGTH`)
- Descriptive file names matching their single responsibility

### Commit Messages
- Conventional commits: `feat:`, `fix:`, `docs:`, `chore:`
- Descriptive but concise (e.g., `feat: add Claude API wrapper with 25s timeout and JSON parsing`)

## Key Patterns

- **Fail-open rate limiting:** If Upstash is misconfigured, requests are allowed through so the estimator keeps working during setup.
- **Lazy singleton rate limiter:** `getLimiter()` in `ratelimit.js` creates the Upstash client once on first request and reuses it across invocations.
- **25s timeout via `Promise.race`:** Stays within Vercel's 30s function limit. Throws typed `api_timeout` error.
- **XSS prevention:** Frontend uses `escHtml()` to sanitize all user-provided text before rendering.
- **JSON-only Claude responses:** System prompt instructs Claude to return structured JSON matching the response schema above. Model: `claude-sonnet-4-6`, `max_tokens: 1024`.
- **Example templates:** Clickable chips pre-fill the textarea with common project descriptions (unit turns, kitchen/bath remodel, roofing, painting).
- **Estimate history:** Saved to `localStorage` (key: `fishbeck_estimates`, max 10 entries). Shown on the input screen so users can revisit past estimates.
- **Smart proposal email:** The "Request My Proposal" CTA pre-fills the email body with the formatted estimate so Jimmy receives full context.
- **Print/copy buttons:** Print opens browser print dialog with clean `@media print` styles. Copy formats estimate as plain text for clipboard.
- **Rotating loading messages:** Cycles through 5 progressive messages every 2.5s during the API call.
- **Accessibility:** Skip-to-content link, focus management on results, shake animation on empty submit, JSON-LD structured data.
- **Error retry:** "Try Again" re-submits the same input instead of resetting to blank form.

## Updating Pricing

All pricing ranges and service categories live in `lib/prompt.js`. Edit the `PRICING REFERENCE` section in `buildSystemPrompt()` to change cost ranges. Push to `master` to auto-deploy.
